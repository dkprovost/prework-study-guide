HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The use of h1, h2, ..., h6 provides differeint size titles.
  • The use of p represents paragraph or block of text, used extensively to wrap most of the text on webpages
  • Lists can be created using ul for unordered and ol for ordered
  • The use of img represents information about images
    • These can contain various attributes
    • The src attribute is most important because it defines location
  • The use of br is for line of empty space or a line break
  • The use of a represents 'ancor'
    • creates links to the same page or other webpages
    • href attributes points to URL for the link
  • The use of header usually contains a navigation menu and/or intro content
    • in our case, it holds h1 heading that renders "Prework Study Guide"
  • The use of main will be discussed later...
  • The use of footer appears at the bottom and usually contains author, copyright, contact, sitemap, and navigation

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • a declaration has two parts - property: value
  • The use of *, a special wild card symbol, declares for all elements on the page
  • The use of link links resources to HTML
    • The use of rel= specifies the relationship between current documents and linked documents
    • The use of href= refers to a location (URL) of external resources

Git

  • Always use git before the command
  • git status: checks what branch we are currently on
  • git checkout: switches to a branch
    • git checkout -b branch-name: creates a new branch 'branch-name'
    • git checkout main: switches back to main branch
  • git pull: pulls latest version from GitHub to local computer
      • git pull origin main: syncs local branch with remote branch
  • git add -A: commits all files in working branch
  • git commit -m "message here": associates message with the commit
  • git push: can create new branch or add new changes to existing branch
    • git push origin feature/branch: pushes changeswe have locally to GitHub branch

JavaScript

  • A variable (var) is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script
  • A for loop will loop a code based on conditionals
  • A value is considered:
    • falsy if one of the following values: 0, -0, On, " ", null, undefined, NaN, or the Boolean value false
    • truthy if all others
  • The code === checks to see if two values are equals
    • If they are, it returns 'true'
    • If they aren't, it returns 'false'
  • length is a built in code for the number of values in array
  • Use of var x=0, x<'function'.length, x++ tells loop to start at beginning of array and go to the end, one at a time.
  • Use of console.log() displays content in console tab in 'insepct' option when right-clicking on browser
